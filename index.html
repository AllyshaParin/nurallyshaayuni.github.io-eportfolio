Lab Work

Hands-On
Learning Labs

Lab 1

User Interface Design

Explored the fundamentals of Android UI layout using XML, crafting responsive and visually structured screens that follow material principles.

Reflection This lab introduced me to the importance of intentional visual hierarchy and spacing in UI design. I gained confidence working with ConstraintLayout and learned how small design decisions have a big impact on user experience. It was rewarding to see a wireframe come to life as a functioning screen.
Lab 2

Interaction Design

Implemented user interaction patterns — button click listeners, input validation, and dynamic UI updates — to make the app feel responsive and intuitive.

Reflection Handling user events taught me to think from the perspective of someone unfamiliar with the app's logic. I learned to provide clear feedback for every action, which is a core principle of good UX. This lab deepened my understanding of the relationship between code and user behaviour.
Lab 3

Material Design

Applied Google's Material Design guidelines to build polished, consistent UI components including cards, FABs, bottom sheets, and theming with colour palettes.

Reflection Material Design gave me a framework for making confident design decisions backed by established principles. I appreciated how theming with a single colour token propagates throughout the entire UI. Applying elevation and shape to components made the app look professional and trustworthy.
Lab 4

Navigation

Built multi-screen navigation flows using the Android Navigation Component, including a bottom navigation bar, fragment back stack management, and safe args.

Reflection Navigation architecture changed how I think about structuring an app — each screen is a destination, not just a class. Managing the back stack gracefully was more nuanced than I expected, and this lab prepared me well for building the PathWays project. I now design navigation flows before writing a single line of code.
Lab 5

Room Database

Integrated Android Room for local data persistence — defining entities, DAOs, and a database class to store and query structured data offline.

Reflection Room abstracted the complexity of SQLite while keeping me in control of the schema. Using LiveData with the DAO made the UI reactive, automatically reflecting database changes without manual refresh calls. This lab was a turning point in understanding offline-first architecture — crucial for apps targeting low-connectivity communities.

Capstone Projects

PathWays —
Built to Matter

Project 1 · Lab 4 + Working Features

PathWays

Problem: Individuals living in poverty often lack accessible guidance on government aid programmes and employment pathways available to them. PathWays bridges this information gap through a structured, easy-to-navigate mobile application.

Lessons Learned Building a full navigation-driven app taught me the value of planning screen flows on paper before touching code. I encountered real challenges managing fragment state across the back stack and learned to use the Navigation Graph as a single source of truth. Most importantly, I realised that a clean UI is meaningless without content that genuinely addresses a user's real-world problem.

Project 2 · Lab 5 + API + Sensors + Cloud

PathWays v2

Problem: The first version relied on static content; users needed real-time information on job listings and aid eligibility. Version 2 integrates live APIs, device sensors, and cloud sync to deliver personalised, up-to-date guidance.

Lessons Learned Integrating Retrofit for REST APIs and Firebase Firestore simultaneously was challenging but deeply instructive. I learned to handle asynchronous data flows with Kotlin coroutines and to structure a repository layer so that the UI never cares whether data comes from the network or the local Room cache. Using CameraX for document scanning opened my eyes to how sensors can reduce friction for users with limited digital literacy.

Learning Journey

Looking Back,
Moving Forward

"Technology is most powerful when it serves those who need it most."

This course has been one of the most transformative experiences of my academic journey. Starting from scratch with Android development, I progressed from building simple XML layouts to architecting a full-stack mobile application with live APIs, cloud storage, and on-device sensors — all within a single semester.


What surprised me most was how deeply design and engineering are intertwined. A button's placement, a loading state's duration, a database query's efficiency — every decision affects whether a real person can use the app effectively or gives up in frustration. Connecting my work to SDG 1 kept that human dimension present throughout every lab and project.


I leave this course with not just technical skills, but a clearer sense of purpose: to build software that is inclusive, accessible, and genuinely useful to communities that technology has historically overlooked. I am grateful to Cikgu Izwan for the structured yet exploratory approach to teaching that made this growth possible.
